# Механика этапов EduCase

## Шесть фиксированных этапов (без редактора графа узлов)
1. **Пациенты** — опциональный контекстный поиск по ключевым словам + карточки пациентов.
2. **Клинико-эпидемиологический диагноз** — поиск + точка ветвления + документы (ДМ4, форма 23).
3. **Обследование контактных лиц** — схема объекта с персоналом + иконки действий.
4. **Обследование объектов внешней среды** — схема + фото + иконки действий + документы
   (акт ГСЭН, донесение командира с ловушками-обманками).
5. **Оценка СЭО** — поиск + документы (план + приказ с обманками).
6. **Окончательный эпидемиологический диагноз** — поиск + документы (акт расследования вспышки)
   + таймлайны наблюдения.

## Ветвление «Вариант B» (этап 2)
Оба пути (верный/неверный) проходят одни и те же этапы. Ошибка **не блокирует** прохождение и
вскрывается только в финальном отчёте.

## Поиск по ключевым словам
Строгое сопоставление с синонимами, которые задаёт преподаватель. **Никакого fuzzy-матчинга,
никакой устойчивости к опечаткам.**

## Механика документов («Вариант B»)
Курсант (а) выбирает правильный документ из списка с обманками И (б) заполняет его поля;
поля сверяются с ожидаемыми значениями. Детали полей — в `educase-document-templates`.

## Осмотр помещений (виртуальный обход)
Двухуровневая навигация: план этажа с кликабельными входами → интерьерные фото с хотспотами
→ свободный вывод курсанта, проверяемый по ключевым словам преподавателя. Встречается на
этапах 3 и 4 — входит в MVP.
